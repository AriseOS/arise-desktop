# 产品愿景 V1

> 本文档记录产品方向讨论的结论，作为后续设计和开发的指导。

---

## 1. 产品定位

### 核心定位：分身

**不是工具，是另一个我。**

| 维度 | 工具思维 | 分身思维 |
|-----|---------|---------|
| 关系 | 我用它 | 它是我 |
| 能力来源 | 产品预设 | 从我身上学 |
| 使用方式 | 我教它做 | 它看我做，然后会了 |
| 价值感知 | "这个功能好用" | "它懂我" |

### 差异化

- **vs ChatGPT**：不是更聪明的 AI，是懂我的 AI
- **vs RPA**：不是写脚本，是学习我的方式
- **vs Rewind/Recall**：不只是搜索记忆，是能替我做事

---

## 2. 第一阶段定位

### 聚焦场景：数据收集整理

选择原因：
- 对准确性要求高，能体现"学会用户方式"的价值
- 高频刚需，早期用户感知强

### Memory 策略

| 用户类型 | 录制方式 | 价值 |
|---------|---------|------|
| 核心用户 | 全量录制 | 完整学会用户的所有工作方式 |
| 普通用户 | 主动录制 | 录一段、分享一段 |
| 公共价值 | - | 后台可见总结出的方法论 |

### 分身边界（第一阶段）

```
✗ 不主动做事
✗ 不后台做事
✓ 用户对话发起 → 分身执行
```

## 3. 交互流程

### 基本流程

```
用户发起任务（自然语言）
    ↓
Ami 决定怎么做
    ├─ 有 Memory → "根据你 X 月的做法，我会..."
    ├─ 有 Public → "我用了 @xxx 分享的方法..."
    └─ 都没有   → "我尝试这样做..."
    ↓
执行（一般不确认，直接做）
    ↓
结果呈现
    ├─ 对话中简要说明（含技能来源）
    └─ 生成文档/Excel + 打开链接
```

### 交互原则

| 原则 | 说明 |
|-----|------|
| 不啰嗦 | 一般不确认，直接做 |
| 有依据时说明 | 有 Memory 参考时告诉用户 |
| 透明来源 | 用了 Public 技能时说明来自谁 |
| 结果要可用 | 生成文件而非纯文本 |

## 4. Memory 设计理念

### 二维结构

**纵轴：抽象层级**

| 层级 | 说明 | 存储方式 |
|-----|------|---------|
| 原始事件 | 最细粒度的交互 | 不存 |
| 操作序列 | 页面内的一组操作 | IntentSequence |
| 任务 | 完整的工作单元 | CognitivePhrase（打散存入图） |
| 方法 | 做某类事的通用模式 | 从图的路径中涌现 |
| 偏好 | 用户的习惯倾向 | 从图的统计中计算 |

**横轴：Domain（来源）**

| Domain | 说明 | 第一阶段 |
|-------|------|---------|
| 网页浏览 | 浏览器内的操作 | ✓ 先做 |
| 文档编辑 | 本地文档操作 | 后续 |
| 某个应用 | 特定应用内的操作 | 后续 |

### 核心理念

**方法和偏好不是单独存的实体，而是从数据中涌现的。**

```
CognitivePhrase（一次完整任务）
    ↓ 打散
State + IntentSequence + Action
    ↓ 写入图
    ↓ 多次录制后积累统计
偏好 = 某个 State 上哪些操作做得多
方法 = 图上的高频路径
```

### 存储形式：图

```
节点：State（页面状态）
边：Action（页面跳转）
节点属性：IntentSequence 列表 + 访问统计
边属性：触发方式 + 使用频率
```

图能表达：
- 单点能力：某个页面能做什么
- 路径方法：怎么从 A 到 B
- 频率偏好：哪条路走得多

### Domain 扩展

每个 Domain 是一张独立的图，未来可能需要跨 Domain 连接。

## 5. 产品功能定义

### 功能模块

```
┌─────────────────────────────────────────────────────────────┐
│                         Ami 分身                             │
├─────────────┬─────────────┬─────────────┬──────────────────┤
│   录制学习   │   对话交互   │   任务执行   │    结果管理      │
└─────────────┴─────────────┴─────────────┴──────────────────┘
```

### 1. 录制学习

| 功能 | 说明 |
|-----|------|
| 全量录制 | 后台持续记录浏览器操作（核心用户） |
| 主动录制 | 用户手动开始/结束一段录制（普通用户） |
| 录制回看 | 查看录制的内容，确认分身学到了什么 |
| 录制管理 | 删除不想让分身学的内容 |

### 2. 对话交互

| 功能 | 说明 |
|-----|------|
| 自然语言输入 | 用对话方式描述任务 |
| 任务理解确认 | 分身理解后简要说明"我会这样做" |
| 执行中状态 | 展示分身正在做什么 |
| 结果呈现 | 任务完成后展示结果摘要 |

### 3. 任务执行

| 功能 | 说明 |
|-----|------|
| 基于 Memory 执行 | 按用户之前的方式做 |
| 基于 Public 执行 | 使用社区分享的技能 |
| 无依据尝试 | 没见过的任务，尝试去做 |
| 执行中断/调整 | 用户可以中途干预 |

**第一阶段任务类型**：
- 网页数据采集
- 信息整理
- 竞品调研

### 4. 结果管理

| 功能 | 说明 |
|-----|------|
| 结果文件生成 | 自动生成 Excel/文档 |
| 结果预览 | 在对话中展示摘要 |
| 文件打开/导出 | 一键打开生成的文件 |
| 历史记录 | 查看之前做过的任务和结果 |

### 功能优先级

| 优先级 | 功能 | 理由 |
|-------|------|------|
| P0 | 录制（全量/主动） | 没有录制就没有学习 |
| P0 | 对话交互 | 用户发起任务的唯一入口 |
| P0 | 任务执行 | 核心价值 |
| P0 | 结果文件生成 | 任务产出 |
| P1 | 执行过程可见 | 建立信任 |
| P1 | 历史记录 | 找回之前的结果 |
| P2 | Memory 可视化 | 理解分身能力 |
| P2 | 录制管理 | 精细控制学习内容 |

## 6. Public 社区

### 定位

Public 是技能社区，让分身能力超越"我自己会的"。

### 用户行为

| 行为 | 说明 |
|-----|------|
| 浏览 | 看热门/最新的技能 |
| Share | 把自己的技能分享出去 |
| Like | 给别人的技能点赞 |

### 激励机制

| 指标 | 奖励 |
|-----|------|
| 被 Like 最多 | Token 奖励 |
| 被使用最多 | Token 奖励 |

### 使用方式

**不需要用户主动"学会"**：
- 用户发起任务
- 分身先查自己的 Memory
- 没有就自动查 Public
- 找到合适的就用，并告知用户来源

### Share 粒度

不限定粒度，用户用了什么就能分享什么：
- 完整任务（CognitivePhrase）
- 方法（多个任务聚合）
- 单点操作（State + IntentSequence）

系统自动打标签，记录使用和反馈。

### 管理方式

```
Memory：我的技能（我录制的）
Public：社区技能（大家分享的）

两者完全分开管理。
```

## 7. UI/UX 设计

### 设计理念

**分身，不是工具面板。**

参考 Sky.app 的设计思路：轻量、浮动、融入系统、随叫随到。

### 两层交互结构

| 层级 | 形态 | 用途 |
|-----|------|-----|
| **日常交互** | 浮动小圆点 + 轻量对话框 | 快速发起任务、看结果 |
| **深度使用** | Dashboard（完整界面） | 管理 Memory、浏览 Public、查看历史 |

### 主界面：浮动小圆点

```
┌─────────────────────────────────────────┐
│                                         │
│            （用户桌面）                  │
│                                         │
│                              ┌───┐      │
│                              │ ● │ ← 小圆点，常驻
│                              └───┘      │
│                                         │
└─────────────────────────────────────────┘
```

**交互方式**：
- 点击/快捷键 → 呼出对话输入框
- 对话在浮动窗口中完成
- 需要时展开成完整 Dashboard

**设计原则**：
- 像系统的一部分，不是一个"App"
- 随时可用，低打扰
- 颜色融入系统，感觉是原生增强

### Dashboard：三个 Tab

```
┌─────────────────────────────────────────┐
│  ┌──────┐  ┌──────┐  ┌──────┐          │
│  │ 对话 │  │Memory│  │Public│    ● 录制中│
│  └──────┘  └──────┘  └──────┘          │
├─────────────────────────────────────────┤
│                                         │
│              Tab 内容区                  │
│                                         │
├─────────────────────────────────────────┤
│  [输入...]                      [发送]  │
└─────────────────────────────────────────┘
```

**三个 Tab**：

| Tab | 功能 | 用户心智 |
|-----|------|---------|
| **对话** | 发起任务、看结果、历史任务入口 | "让分身帮我做事" |
| **Memory** | 查看分身学会了什么，可管理 | "看看它会什么，删掉不想要的" |
| **Public** | 浏览社区技能、分享自己的技能 | "学别人的 / 分享我的" |

### 用户心智

- 日常：小圆点常驻，随时呼唤 → 贾维斯感
- 深入：打开 Dashboard 管理和浏览 → 实用性

## 8. 用户旅程

### 首次启动流程

```
下载安装
    ↓
首次打开 → 权限申请（屏幕录制、辅助功能等）
    ↓
模式选择
    ↓
进入 Dashboard
    ↓
小圆点常驻桌面
```

### 模式选择

| 英文 | 中文 | 说明 | 状态 |
|-----|------|------|------|
| Coach Mode | **普通模式** | 用户主动录制，边做边教 | 可选 |
| Shadow Mode | **如影模式** | 全量录制，静默学习 | 灰置，即将开放 |

**模式选择弹窗**：

```
┌─────────────────────────────────────────┐
│                                         │
│         选择学习模式                     │
│                                         │
│  ┌───────────────┐  ┌───────────────┐  │
│  │   普通模式     │  │   如影模式    │  │
│  │  Coach Mode   │  │  Shadow Mode  │  │
│  │               │  │               │  │
│  │  你教，它学    │  │  它看，它学   │  │
│  │               │  │               │  │
│  │    [开始]     │  │   即将开放    │  │
│  └───────────────┘  └───────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

### 小圆点交互

**默认行为：录屏优先**

```
点击小圆点
    ↓
弹出输入框
    ├─ 用户输入文字 → 发起对话/任务
    └─ 用户不输入 → 默认开始录屏
```

不需要用户单独选择"录制"或"对话"，输入即对话，不输入即录制。

### 任务结果呈现

**两层展示**：

```
浮窗（摘要）
┌─────────────────────────────────────┐
│ ✓ 查完了，3 个竞品的定价对比         │
│ [查看结果]                          │
└─────────────────────────────────────┘
        ↓ 点击
Dashboard（完整渲染）
```

**文本内容**：Markdown 渲染（前端已支持 ReactMarkdown）

**文件类型分层渲染**：

| 文件类型 | 展示方式 | 理由 |
|---------|---------|------|
| 图片 (png, jpg, gif, webp, svg) | 直接渲染缩略图 | 用户可立即查看，点击可放大 |
| HTML | 直接渲染（iframe 沙箱） | Agent 生成的报告可直接预览 |
| PDF | 缩略图 + 点击打开 | PDF 嵌入体验差，用系统应用打开 |
| CSV/Excel | 表格预览（前几行） | 展示数据结构，点击完整打开 |
| 代码文件 | 语法高亮预览 | 方便查看生成的代码 |
| 文件夹 | 文件列表 + 点击跳转 Finder | 无法直接渲染，需系统打开 |
| 其他文档 (docx, pptx) | 文件卡片 + 点击打开 | 无法嵌入渲染 |

### 日常使用流程

```
小圆点常驻桌面
    ↓
点击小圆点
    ├─ 想做任务 → 输入文字 → 分身执行 → 返回结果
    └─ 想教它 → 不输入 → 开始录制 → 完成后自动学习
    ↓
需要深入管理 → 展开 Dashboard
    ├─ 对话 Tab：任务历史
    ├─ Memory Tab：管理技能
    └─ Public Tab：浏览/分享
```

## 9. 冷启动方案

### 预置 Public Memory

新用户没有自己的 Memory，通过 **官方预置的基础技能** 解决冷启动：

| 类型 | 示例 |
|-----|------|
| 常用网站操作 | Google 搜索、Wikipedia 查资料 |
| 数据收集 | 从网页提取表格、批量获取信息 |
| 常见工作流 | 竞品调研、信息汇总 |

### 用户感知

```
新用户："帮我查一下 XX 公司的信息"
    ↓
Ami 没有用户的 Memory
    ↓
自动使用 Public 中的预置技能
    ↓
"我用官方提供的方法帮你查，查完了..."
```

**用户第一次用就能干活**，不会感觉"它什么都不会"。

### 逐步过渡

```
第 1 天：100% 依赖 Public
    ↓
用户开始录制 / 使用
    ↓
第 N 天：Memory 积累，Public 依赖减少
    ↓
最终：主要用自己的 Memory，Public 作为补充
```
