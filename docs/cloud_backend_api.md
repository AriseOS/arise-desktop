# Cloud Backend API Documentation

## Base URL

```
http://localhost:9000
```

## Overview

Cloud Backend provides AI-powered workflow generation from user recordings:
1. Upload recordings → Extract intents → Build Intent Memory Graph (per user)
2. Generate MetaFlow from Intent Graph based on task description
3. Generate Workflow YAML from MetaFlow

## Key Concepts

- **Intent Memory Graph**: Cumulative knowledge base of user intents. Each user has ONE graph that grows with each recording.
- **MetaFlow**: Intermediate representation between Intent Graph and Workflow. Generated by filtering relevant intents based on task description.
- **Workflow**: Executable YAML definition for automation agents.

## Recording APIs

### Upload Recording

**POST** `/api/recordings/upload`

Upload user operation recordings with task description. Backend will:
1. Save recording to filesystem
2. Extract intents in background (async) using the task_description
3. Add intents to user's Intent Memory Graph

**Request Body:**
```json
{
  "user_id": "user123",
  "task_description": "Search for coffee on Google",
  "operations": [
    {
      "type": "click",
      "url": "https://www.google.com",
      "element": {"tag": "INPUT", "name": "q"},
      "timestamp": "2025-11-08 16:06:10"
    }
  ]
}
```

**Response:**
```json
{
  "recording_id": "uuid-here"
}
```

**Notes:**
- `task_description`: User's description of what they did (used by LLM to extract better intents)
- Returns immediately (intent extraction happens in background)
- Intents are added to `~/.ami/users/{user_id}/intent_graph.json`
- Each recording adds more intents to the same graph (cumulative)
- The task_description helps the LLM understand user intent and generate more accurate intents

## MetaFlow APIs

### Generate MetaFlow

**POST** `/api/users/{user_id}/generate_metaflow`

Generate MetaFlow from user's Intent Memory Graph based on task description.
MetaFlowGenerator filters relevant intents from the graph.

**Path Parameters:**
- `user_id` - User ID

**Request Body:**
```json
{
  "task_description": "Search for coffee on Google"
}
```

**Response:**
```json
{
  "metaflow_id": "metaflow_abc123xyz",
  "metaflow_yaml": "name: metaflow_abc123xyz\nnodes:\n  - ...",
  "task_description": "Search for coffee on Google",
  "status": "success"
}
```

**Notes:**
- Synchronous operation (blocks until complete, 30-60 seconds)
- Uses LLM to filter relevant intents and generate MetaFlow
- Returns MetaFlow YAML directly in response (no need for separate download)
- Also saves MetaFlow to server at `~/.ami/users/{user_id}/metaflows/{metaflow_id}/`
- Requires user to have uploaded recordings first (Intent Graph must exist)

**Errors:**
- 400: Missing task_description
- 404: User has no Intent Graph (no recordings uploaded yet)
- 500: MetaFlow generation failed

## Workflow APIs

### Generate Workflow from MetaFlow

**POST** `/api/metaflows/{metaflow_id}/generate_workflow`

Generate Workflow YAML from MetaFlow.

**Path Parameters:**
- `metaflow_id` - MetaFlow ID

**Request Body:**
```json
{
  "user_id": "user123"
}
```

**Response:**
```json
{
  "workflow_name": "workflow_20251108_160557",
  "workflow_yaml": "name: workflow_20251108_160557\nsteps:\n  - ...",
  "status": "success"
}
```

**Notes:**
- Synchronous operation (blocks until complete, 30-60 seconds)
- Uses LLM to generate Workflow YAML from MetaFlow
- Returns Workflow YAML directly in response (no need for separate download)
- Also saves workflow to server at `~/.ami/users/{user_id}/workflows/{workflow_name}/`

**Errors:**
- 400: Missing user_id
- 404: MetaFlow not found
- 500: Workflow generation failed

## Complete Flow Example

### 1. Upload Recording (with Intent Extraction)

```bash
curl -X POST http://localhost:9000/api/recordings/upload \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "default_user",
    "task_description": "Search for coffee on Google",
    "operations": [...]
  }'
```

Response:
```json
{
  "recording_id": "abc-123-def"
}
```

Backend will extract intents (using task_description) and add to user's Intent Graph in background.

### 2. Generate MetaFlow

```bash
curl -X POST http://localhost:9000/api/users/default_user/generate_metaflow \
  -H "Content-Type: application/json" \
  -d '{
    "task_description": "Search for coffee on Google"
  }'
```

Response:
```json
{
  "metaflow_id": "metaflow_xyz789",
  "status": "success"
}
```

### 3. Generate Workflow

```bash
curl -X POST http://localhost:9000/api/metaflows/metaflow_xyz789/generate_workflow \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "default_user"
  }'
```

Response:
```json
{
  "workflow_name": "workflow_20251108_160557",
  "status": "success"
}
```

### 4. Download Workflow

```bash
curl "http://localhost:9000/api/workflows/workflow_20251108_160557/download?user_id=default_user"
```

Response:
```json
{
  "yaml": "name: workflow_20251108_160557\n..."
}
```

## Data Storage

All data is stored in the server filesystem:

```
~/.ami/users/{user_id}/
├── intent_graph.json          # User's Intent Memory Graph (cumulative)
├── recordings/                # All recordings
│   └── {recording_id}/
│       └── operations.json
├── metaflows/                 # Generated MetaFlows
│   └── {metaflow_id}/
│       ├── metaflow.yaml
│       └── task_description.txt
└── workflows/                 # Generated Workflows
    └── {workflow_name}/
        ├── workflow.yaml
        └── metaflow.yaml
```

Default base path: `~/.ami/`

## Key Features

### Cumulative Intent Graph
- Each user has ONE Intent Memory Graph
- Every recording adds more intents to the same graph
- Graph grows and improves over time

### Smart Intent Filtering
- MetaFlow generation filters relevant intents based on task description
- Uses LLM to understand semantic similarity
- Only relevant intents are included in the MetaFlow

### Persistent Workflow Storage
- All generated workflows are saved permanently
- Users can list and download workflows anytime
- Workflows include the MetaFlow they were generated from
