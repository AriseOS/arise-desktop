# MetaFlow Specification

MetaFlow is an intermediate representation between Intent Memory Graph and Workflow. It captures the semantic flow of user intents in a structured YAML format.

## Basic Structure

```yaml
version: "1.0"
task_description: "Task description"

nodes:
  # Regular node
  - id: node_1
    intent_id: intent_xxx
    intent_name: "NavigateToSite"
    intent_description: "Navigate to website"
    operations:
      - type: navigate
        url: "https://example.com"
        element: {}
    outputs:  # Optional: if this node produces output
      output_key: "variable_name"

  # Loop node
  - id: node_2
    type: loop
    description: "Iterate through list, process each item"
    source: "{{list_variable}}"
    item_var: "current_item"
    children:
      - id: node_2_1
        intent_id: intent_yyy
        intent_name: "ProcessItem"
        intent_description: "Process single item"
        operations: [...]
        inputs:  # Optional: use loop variable
          item_url: "{{current_item.url}}"
        outputs:
          result: "item_result"
```

## Node Types

### 1. Regular Node

Direct mapping from an Intent in the Intent Graph.

**Required Fields**:
- `id`: Unique node identifier (e.g., `node_1`, `node_2`)
- `intent_id`: Reference to the original Intent ID
- `intent_name`: Simplified name in PascalCase
- `intent_description`: Human-readable description of what this node does
- `operations`: List of operations from the Intent (full copy)

**Optional Fields**:
- `inputs`: Variables this node consumes
- `outputs`: Variables this node produces

**Example**:
```yaml
- id: node_1
  intent_id: intent_a3f5b2c1
  intent_name: "NavigateToAllegro"
  intent_description: "Navigate to Allegro homepage"
  operations:
    - type: navigate
      url: "https://allegro.pl"
      page_title: "Allegro"
```

### 2. Loop Node

For iterating over a list of items.

**Required Fields**:
- `id`: Unique node identifier
- `type`: Must be `loop`
- `description`: Natural language description of the loop
- `source`: Data source variable (e.g., `{{product_urls}}`)
- `item_var`: Name of the loop variable
- `children`: List of child nodes to execute for each item

**Example**:
```yaml
- id: node_3
  type: loop
  description: "Iterate through product list, extract detailed info"
  source: "{{product_urls}}"
  item_var: "current_product"
  children:
    - id: node_3_1
      intent_id: intent_b4c6d2e3
      intent_name: "ExtractProductDetail"
      intent_description: "Extract product details"
      inputs:
        product_url: "{{current_product.url}}"
      operations:
        - type: navigate
          url: "{{current_product.url}}"
        - type: extract
          target: "title"
          element:
            xpath: "//h1[@class='title']"
      outputs:
        product_info: "product_info"
```

### 3. Inferred Node

Nodes generated by analyzing the gap between recorded operations and user query. These are marked with `(Inferred)` in the intent_description.

**Types of Inferred Nodes**:

**Text Processing Node**:
```yaml
- id: node_inferred_process
  intent_id: inferred_text_process
  intent_name: "TranslateData"
  intent_description: "Translate extracted data to Chinese (Inferred)"
  operations:
    - type: text_process
      params:
        source: "{{extracted_data}}"
  outputs:
    translated_data: "translated_data"
```

**Autonomous Task Node**:
```yaml
- id: node_inferred_autonomous
  intent_id: inferred_autonomous
  intent_name: "FindCEOLinkedIn"
  intent_description: "Find CEO LinkedIn profile (Inferred)"
  operations:
    - type: autonomous_task
      params:
        goal: "Find the CEO's LinkedIn profile"
  outputs:
    result: "ceo_linkedin"
```

## Data Flow

### Variable References

Use `{{variable_name}}` syntax to reference variables:

- Simple reference: `{{product_urls}}`
- Field access: `{{product_info.title}}`
- Array access: `{{items[0]}}`

### Outputs

Nodes that produce data should declare their outputs:

```yaml
outputs:
  variable_name: "variable_name"
```

The key and value are typically the same. The output becomes available for subsequent nodes.

### Inputs

Nodes can declare their inputs for clarity:

```yaml
inputs:
  product_url: "{{current_product.url}}"
```

## Operation Types

Operations in MetaFlow come directly from the recorded user operations:

| Type | Description | Key Fields |
|------|-------------|------------|
| `navigate` | Navigate to URL | `url`, `page_title` |
| `click` | Click an element | `element.xpath`, `element.href` |
| `input` | Enter text | `element.xpath`, `data.inputValue` |
| `extract` | Extract data | `target`, `element.xpath`, `value` |
| `scroll` | Scroll page | `data.direction`, `data.distance` |
| `select` | Select text | `element.xpath`, `data.selectedText` |
| `copy_action` | Copy text | `data.copiedText` |
| `store` | Store data | `params.collection`, `params.fields` |
| `text_process` | LLM text processing | `params.source` |
| `autonomous_task` | Autonomous browser task | `params.goal` |

## Intent Name Generation

Generate PascalCase names from intent descriptions:

- "Navigate to Allegro homepage" → `NavigateToAllegro`
- "Enter coffee category page" → `EnterCoffeeCategory`
- "Extract product info" → `ExtractProductInfo`
- "Translate to Chinese (Inferred)" → `TranslateToChiense`

## Node Ordering

Nodes should be ordered logically:

1. Navigation nodes first
2. Extraction nodes after navigation
3. Inferred processing nodes after the data they process
4. Loop nodes contain their children
5. Storage nodes typically at the end or inside loops

## Complete Example

```yaml
version: "1.0"
task_description: "Collect all coffee products from Allegro and translate to Chinese"

nodes:
  - id: node_1
    intent_id: intent_a1b2c3d4
    intent_name: "NavigateToAllegro"
    intent_description: "Navigate to Allegro homepage"
    operations:
      - type: navigate
        url: "https://allegro.pl"
        page_title: "Allegro"

  - id: node_2
    intent_id: intent_e5f6g7h8
    intent_name: "NavigateToCoffeeCategory"
    intent_description: "Navigate to coffee category page"
    operations:
      - type: click
        url: "https://allegro.pl"
        element:
          xpath: "//a[contains(text(), 'Kawa')]"
          href: "https://allegro.pl/kategoria/kawa"
      - type: navigate
        url: "https://allegro.pl/kategoria/kawa"

  - id: node_3
    intent_id: implicit_extract_list
    intent_name: "ExtractProductList"
    intent_description: "Extract product URL list (inferred node)"
    operations:
      - type: extract
        target: "product_urls"
        element:
          xpath: "//article//a[@class='product-link']"
        value: []
    outputs:
      product_urls: "product_urls"

  - id: node_4
    type: loop
    description: "Iterate through products, collect details and translate"
    source: "{{product_urls}}"
    item_var: "current_product"
    children:
      - id: node_4_1
        intent_id: intent_i9j0k1l2
        intent_name: "ExtractProductDetail"
        intent_description: "Extract product title and price"
        inputs:
          product_url: "{{current_product.url}}"
        operations:
          - type: navigate
            url: "{{current_product.url}}"
          - type: extract
            target: "title"
            element:
              xpath: "//h1[@class='product-title']"
            value: "Sample Product"
          - type: extract
            target: "price"
            element:
              xpath: "//span[@class='price']"
            value: "99.99 zł"
        outputs:
          product_info: "product_info"

      - id: node_4_2
        intent_id: inferred_text_process
        intent_name: "TranslateProduct"
        intent_description: "Translate product info to Chinese (Inferred)"
        operations:
          - type: text_process
            params:
              source: "{{product_info}}"
        outputs:
          translated_info: "translated_info"

      - id: node_4_3
        intent_id: intent_m3n4o5p6
        intent_name: "StoreProduct"
        intent_description: "Store product to database"
        operations:
          - type: store
            params:
              collection: "coffee_products"
              fields: ["title", "price"]
```
