## 1. Current Implementation Analysis

### 1.1 Current Script Generation Flow

**Current Method:** `_generate_extraction_script_with_llm()` (line 848-987)

```python
async def _generate_extraction_script_with_llm(self, dom_analysis, data_requirements, ...):
    # 1. Build prompt with DOM structure + requirements
    prompt = f"""... DOM structure and instructions ..."""

    # 2. Call LLM provider (single-turn)
    response = await self.provider.generate_response(
        system_prompt=self.SYSTEM_PROMPT,
        user_prompt=prompt
    )

    # 3. Extract and wrap code
    return self._extract_and_wrap_code(response)
```

**Characteristics:**
- ✅ Single LLM call, generates script as text string
- ✅ Stores script in KV database as string blob
- ❌ No testing or validation of generated script
- ❌ No iterative refinement if script has errors
- ❌ Hard to debug when script fails

## 4. Proposed Integration Approach

### 4.1 Minimal Changes Approach (Recommended)

**Goal:** Integrate Claude SDK with minimal disruption to existing code

**Changes:**

1. **Add new method:** `_generate_with_claude_sdk()`
2. **Modify:** `_generate_extraction_script_with_llm()` to optionally use Claude SDK
3. **Keep:** All other logic unchanged (script key, caching, execution)

**Implementation:**

```python
async def _generate_extraction_script_with_llm(self, dom_analysis, data_requirements, ...) -> str:
    """Generate extraction script - optionally using Claude SDK for iterative refinement"""

    # Check if Claude SDK should be used
    use_claude_sdk = self.config_service.get("claude_agent.enabled", False) if self.config_service else False

    if use_claude_sdk:
        # Use Claude SDK for multi-turn generation
        return await self._generate_with_claude_sdk(dom_analysis, data_requirements, ...)
    else:
        # Use current single-turn method (default)
        return await self._generate_with_single_turn(dom_analysis, data_requirements, ...)

async def _generate_with_claude_sdk(self, dom_analysis, data_requirements, ...) -> str:
    """Generate script using Claude SDK with iterative refinement"""
    from src.common.llm import ClaudeAgentProvider

    # 1. Create working directory
    script_key = self._generate_script_key(data_requirements)
    scripts_root = self.config_service.get_path("data.scripts")
    working_dir = scripts_root / script_key
    working_dir.mkdir(parents=True, exist_ok=True)

    # 2. Prepare input files
    (working_dir / "requirement.json").write_text(json.dumps(data_requirements, ensure_ascii=False))
    (working_dir / "dom_data.json").write_text(json.dumps(dom_analysis['dom_dict'], ensure_ascii=False))

    # 3. Build Claude SDK prompt
    prompt = self._build_claude_sdk_prompt(data_requirements)

    # 4. Run Claude SDK
    claude_provider = ClaudeAgentProvider(config_service=self.config_service)
    result = await claude_provider.run_task(
        prompt=prompt,
        working_dir=working_dir,
        max_iterations=5
    )

    if not result.success:
        raise RuntimeError(f"Claude SDK script generation failed: {result.error}")

    # 5. Read generated script
    script_path = working_dir / "extraction_script.py"
    if not script_path.exists():
        raise FileNotFoundError("extraction_script.py not generated")

    script_content = script_path.read_text()

    # 6. Wrap with execution wrapper (same as current)
    return self._extract_and_wrap_code(script_content)

async def _generate_with_single_turn(self, dom_analysis, data_requirements, ...) -> str:
    """Current single-turn generation method (preserved as fallback)"""
    # Move existing code here
    llm_view = dom_analysis['llm_view']
    prompt = f"""... existing prompt ..."""

    response = await self.provider.generate_response(
        system_prompt=self.SYSTEM_PROMPT,
        user_prompt=prompt
    )

    return self._extract_and_wrap_code(response)
```

## 6. Implementation Summary

### 6.1 Final Design Decisions

Based on user feedback, the following decisions were made:

1. **Working Directory**: ✅ Independent directory per requirement
   - Path: `data/scripts/{script_key}/`
   - Files: `requirement.json`, `dom_data.json`, `extraction_script.py`

2. **Prompt Construction**: ✅ Clear file-based instructions
   - Prompt explicitly states directory structure and file locations
   - Instructs Claude to use Grep/Read tools to explore DOM
   - Includes testing and validation steps

3. **DOM Transmission**: ✅ JSON file format
   - Saves `dom_dict` as `dom_data.json`
   - Claude can grep/search the file flexibly
   - No token limits (Claude reads as needed)

4. **Script Storage**: ✅ File-based only
   - Removed KV database dependency completely
   - Scripts cached in `{script_key}/extraction_script.py`
   - Breaking change - no backward compatibility with KV storage

5. **Testing Integration**: ✅ Claude SDK handles testing
   - Prompt instructs Claude to create `test_script.py`
   - Claude runs tests and fixes errors iteratively
   - Higher quality scripts with validation

6. **Error Handling**: ✅ Fail fast
   - Raise `RuntimeError` if Claude SDK fails
   - No fallback to single-turn method

7. **Deployment**: ✅ Direct replacement
   - No feature flag - always use Claude SDK
   - One-step migration

#### Key Implementation Details:

**Script Generation Flow:**
```python
1. Generate script_key from data_requirements
2. Create workspace: data/scripts/{script_key}/
3. Save requirement.json (data requirements)
4. Save dom_data.json (DOM structure)
5. Build Claude SDK prompt with:
   - Working directory path
   - File locations
   - DOM structure explanation
   - Extraction requirements
   - Testing instructions
6. Run Claude SDK (max 5 iterations)
7. Read generated extraction_script.py
8. Wrap with execution wrapper
9. Return script content
```

**Script Caching Flow:**
```python
1. Generate script_key from data_requirements
2. Check if {script_key}/extraction_script.py exists
3. If exists: Load from file (cache hit)
4. If not exists: Generate with Claude SDK (cache miss)
5. Execute script with current DOM
```

**File Structure:**
```
~/.local/share/baseapp/scripts/
└── scraper_script_a1b2c3d4/              # script_key hash
    ├── requirement.json                  # Data requirements
    ├── dom_data.json                     # DOM structure
    ├── extraction_script.py              # Generated script
    └── test_script.py                    # Test file (created by Claude)
```

### 6.3 Benefits

1. **Iterative Refinement**: Claude tests and fixes scripts automatically
2. **Better Quality**: Scripts are validated before use
3. **Easy Debugging**: All files accessible in workspace
4. **Flexible DOM Exploration**: Claude can grep/search DOM file
5. **Token Efficiency**: DOM file read on-demand, not sent in full prompt
6. **File-based Cache**: Scripts persist across sessions
7. **Transparency**: All intermediate files visible for inspection

### 6.4 Breaking Changes

- **KV Storage Removed**: Scripts are no longer stored in KV database
- **Migration**: Existing KV-cached scripts will not be loaded
- **Regeneration**: First run after upgrade will regenerate all scripts
