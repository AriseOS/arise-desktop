## 🎯 核心决策

### 1. 产品形态

**最终方案**：**Tauri Desktop App + Chrome Extension + Cloud Backend**

#### 产品组成
```
┌─────────────────────────────────────────┐
│  Tauri Desktop App (主控制中心)          │
│  ├── 嵌入 Python Backend (PyInstaller)  │
│  ├── 完整窗口界面                        │
│  │   ├── Workflow 管理                  │
│  │   ├── 执行控制                        │
│  │   └── 结果展示                        │
│  └── 系统托盘（可后台运行）              │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  Chrome Extension (用户交互界面)         │
│  ├── 录制控制 (开始/停止)                │
│  ├── Workflow 快捷列表                   │
│  ├── 执行触发                            │
│  └── 简单结果展示                        │
│  → 完整的使用体验，保持现有代码          │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  Cloud Backend (数据处理和存储)          │
│  ├── Intent Extraction                  │
│  ├── Intent Graph 构建                   │
│  ├── MetaFlow 生成                       │
│  └── Workflow YAML 生成                  │
└─────────────────────────────────────────┘
```

**关键架构决策**：
- ✅ Extension 不只是录制工具，而是完整的用户交互界面
- ✅ Desktop App 提供更强大的管理和可视化功能
- ✅ 两者可以独立使用，也可以配合使用

**为什么不是纯 Extension？**
- Extension 无法直接控制 Python BaseAgent（需要本地 Backend）
- Extension 权限受限，无法录制桌面应用（未来扩展需求）
- Desktop App 可以提供更好的系统集成和后台运行

**为什么不是纯 Desktop？**
- 用户习惯在浏览器中工作，Extension 更符合用户心智模型
- Extension 可以在真实浏览器环境录制（保留登录状态）
- Extension 提供更流畅的录制和执行体验

### 2. 数据流向与职责划分

**核心原则**：
- 📥 **录制阶段**：完全本地
- ☁️ **处理阶段**：云端分析
- 🖥️ **执行阶段**：本地执行

#### 详细数据流

```
Phase 1: 录制（完全本地）
┌────────────────────────────────────────────┐
│ 1. 用户在 Chrome 浏览器正常操作            │
│ 2. Extension 捕获操作事件                   │
│ 3. 发送到本地 Backend (localhost:8000)     │
│ 4. Backend 保存到本地文件                   │
│    ~/agentcrafter/storage/users/{user_id}/  │
│      learning/{session_id}/                │
│        └── operations.json                 │
│                                            │
│ ✅ 用户可以离线录制                         │
│ ✅ 数据暂不上传云端                         │
└────────────────────────────────────────────┘
              ↓ 用户点击"停止录制"

Phase 2: 上传（后台异步）
┌────────────────────────────────────────────┐
│ 1. 本地 Backend 后台上传 operations.json   │
│ 2. 上传到云端 API                           │
│ 3. 显示上传进度（可选）                     │
│ 4. 上传失败？重试 3 次                      │
│                                            │
│ ✅ 后台进行，不阻塞用户                     │
│ ✅ 本地保留副本（容错）                     │
└────────────────────────────────────────────┘
              ↓ 上传成功

Phase 3: 云端处理（同步等待）
┌────────────────────────────────────────────┐
│ 1. 云端保存 operations.json                │
│ 2. Intent Extraction (调用 LLM)            │
│ 3. 构建 Intent Graph (用户独立)            │
│ 4. 生成 MetaFlow                           │
│ 5. 生成 Workflow YAML                      │
│ 6. 云端保存 workflow.yaml                  │
│                                            │
│ ⏱️ 预计耗时：30-60 秒                       │
│ 🔄 用户等待（模态窗口显示进度）             │
└────────────────────────────────────────────┘
              ↓ 生成完成

Phase 4: 下载 & 执行（本地）
┌────────────────────────────────────────────┐
│ 1. 本地 Backend 自动拉取 workflow.yaml      │
│ 2. 保存到本地 workflows/{workflow_name}/   │
│ 3. 提示用户 "Workflow 已生成！"            │
│ 4. 用户点击"执行"（Extension 或 Desktop）   │
│ 5. BaseAgent 在本地浏览器执行               │
│ 6. 执行结果保存本地 + 上传云端（改进模型）  │
│                                            │
│ ✅ 复用全局浏览器会话                       │
│ ✅ 本地执行，保护隐私                       │
└────────────────────────────────────────────┘
```

### 3. 数据共享策略

#### MVP 阶段（第一版本）

**Intent Graph**：
- ❌ **不跨用户共享**
- ✅ 每个用户独立的 Intent Graph
- 📍 存储位置：云端 `users/{user_id}/intent_graph/`

**Workflow**：
- ❌ **不跨用户共享**
- ✅ 云端备份（作为数据资产）
- ✅ 用户可下载到本地执行
- 📍 存储位置：
  - 云端：`users/{user_id}/workflows/{workflow_name}/`
  - 本地：`~/agentcrafter/storage/users/{user_id}/workflows/`

**录制数据**：
- ✅ **必须上传云端**（构建 Intent Graph 的唯一来源）
- ✅ 云端永久保存
- 📍 存储位置：云端 `users/{user_id}/recordings/{session_id}/`

#### 未来规划（V2）

**团队共享**（2B 场景）：
- 组织内成员可以共享 Workflow
- 权限控制（只读、编辑、执行）
- 专家知识放大

**公开市场**（长期生态）：
- 优秀 Workflow 发布到市场
- 其他用户购买/使用
- 社区贡献

### 4. 用户旅程梳理

#### 完整旅程

```
Step 0: 安装
├─ 下载 Ami.dmg / Ami.exe
├─ 安装 Desktop App
├─ 自动安装 Chrome Extension
├─ 注册/登录账号
└─ 完成初始化

Step 1: 录制
├─ 打开 Extension，点击"开始录制"
├─ 在 Chrome 正常操作（如采集 Allegro 数据）
├─ Extension 实时显示捕获的操作数量
├─ 点击"停止录制"
└─ Extension 显示"正在保存..."

Step 2: 生成
├─ 本地 Backend 上传 operations.json 到云端
├─ 云端开始生成（同步等待）
├─ 显示模态窗口：
│   "正在生成 Workflow...
│    ⏳ 分析操作意图 (完成)
│    ⏳ 构建 MetaFlow (进行中)
│    ⏳ 生成 Workflow (等待)"
├─ 生成完成（30-60 秒）
└─ 提示："Workflow 已生成！点击执行"

Step 3: 执行
├─ 用户点击"执行"（在 Extension 或 Desktop App）
├─ 本地 BaseAgent 加载 workflow.yaml
├─ 在全局浏览器会话中执行
├─ 显示执行进度
└─ 显示结果（成功/失败）

Step 4: 复用
├─ 查看 Workflow 列表（Extension 或 Desktop App）
├─ 选择已有 Workflow
├─ 可选：修改参数（V2 功能）
└─ 一键执行
```

#### 已识别的潜在问题（按优先级）

**P0 - 必须解决（否则 MVP 无法工作）**
1. Backend 未启动 → Extension 无法连接
2. 网络断开 → 无法上传/下载
3. LLM API 调用失败 → 无法生成 Workflow
4. 浏览器会话未初始化 → 执行崩溃
5. 页面结构变化 → 执行失败

**P1 - 应该解决（体验很差）**
6. 端口 8000 被占用 → 自动换端口
7. Extension 安装失败 → 手动安装指引
8. 用户未登录目标网站 → 执行前检测 + 提示
9. 上传/下载失败 → 重试机制
10. Workflow 验证失败 → 友好错误提示

**P2 - 可暂缓（边缘情况）**
11. iframe 跨域问题 → 文档说明限制
12. Shadow DOM → 暂不支持
13. 验证码 → 提示用户手动处理
14. 用户录制中关闭标签页 → 提示保存草稿
15. 磁盘空间不足 → 系统级错误，无需特殊处理

**容错策略**：
- ✅ 不过度设计
- ✅ 做好基础重试和友好提示
- ✅ 记录所有已知问题，逐个解决

### 5. MVP 功能范围界定

#### MVP 必须有（P0）

**录制功能**：
- ✅ 基础操作录制（点击、输入、导航）
- ✅ 录制状态显示
- ✅ 本地存储

**生成功能**：
- ✅ Intent Extraction
- ✅ Intent Graph 构建（每用户独立）
- ✅ MetaFlow 生成
- ✅ Workflow YAML 生成
- ✅ 自动命名（可修改）

**执行功能**：
- ✅ 一键执行 Workflow
- ✅ 执行进度显示
- ✅ 结果展示
- ✅ 错误提示

**管理功能**：
- ✅ Workflow 列表
- ✅ 删除 Workflow
- ✅ 查看 Workflow 详情

**基础设施**：
- ✅ 用户注册/登录
- ✅ 云端数据同步
- ✅ 全局浏览器会话管理

#### V2 计划（P1）

- ⏱️ Workflow 参数化（改变输入值复用）
- ⏱️ Intent Graph 可视化
- ⏱️ Workflow 编辑功能
- ⏱️ 执行失败智能修复
- ⏱️ 定时执行
- ⏱️ 执行历史记录和分析

#### 未来计划（P2）

- 🔮 团队共享（2B）
- 🔮 Workflow 市场
- 🔮 桌面应用录制
- 🔮 跨浏览器支持
- 🔮 云端执行选项
- 🔮 Workflow 组合（Allegro + Amazon → 对比报告）

#### 明确暂不考虑（MVP）

- ❌ 敏感数据处理（加密、脱敏）
- ❌ Cookie/Session 迁移
- ❌ 跨设备同步
- ❌ 多语言支持
- ❌ 高级容错和重试逻辑

### 6. 关键技术决策

#### 客户端技术栈

**Desktop App**：
- 框架：**Tauri** (Rust + Web UI)
  - 理由：比 Electron 更轻量，性能更好
  
**嵌入 Python Backend**：
- 打包：**PyInstaller**
  - 理由：将 Python 代码打包成独立可执行文件
  
**Chrome Extension**：
- 标准：**Manifest V3**
  - 理由：Google 强制迁移，必须使用

#### 云端技术栈

**Backend Framework**：
- **FastAPI** (已有)
  - 理由：高性能，异步支持，自动文档

**数据库**：
- **PostgreSQL** (生产)
- **SQLite** (开发)

**存储**：
- **文件系统**（短期）
- **对象存储**（长期，如 S3）

#### 浏览器控制

**全局单例模式**：
- ✅ BrowserSessionManager 单例
- ✅ 启动时创建，所有 workflow 复用
- ✅ 成本优化：$0.02-0.05/task (vs $1-2)

#### Extension UI 设计

**保持简洁**：
```
┌──────────────────────────┐
│  Ami                     │
├──────────────────────────┤
│  🔴 开始录制              │
│  📋 我的 Workflows (3)    │
│  ⚙️  设置                 │
└──────────────────────────┘

录制中：
┌──────────────────────────┐
│  Ami - 录制中             │
├──────────────────────────┤
│  ⏹️  停止录制 (12 操作)   │
│  📊 实时预览              │
└──────────────────────────┘

Workflow 列表：
┌──────────────────────────┐
│  我的 Workflows           │
├──────────────────────────┤
│  ▶️ 从 Allegro 抓取咖啡   │
│  ▶️ Amazon 产品搜索       │
│  ▶️ 导出数据到 Excel      │
│  + 新建录制               │
└──────────────────────────┘
```

#### Desktop App UI 设计

**完整窗口应用**：
```
┌─────────────────────────────────────────────┐
│  Ami - Workflow Manager                     │
├─────────────┬───────────────────────────────┤
│ Workflows   │  从 Allegro 抓取咖啡产品数据   │
│             │                               │
│ ▶ 电商数据   │  📝 描述：                     │
│   - Allegro │  自动搜索、筛选、抓取           │
│   - Amazon  │                               │
│             │  📊 统计：                     │
│ ▶ 报告生成   │  - 执行次数：5                 │
│             │  - 成功率：100%                │
│ ⚙️ 设置      │  - 最后执行：2 分钟前           │
│             │                               │
│             │  [▶️ 执行] [✏️ 编辑] [🗑️ 删除] │
└─────────────┴───────────────────────────────┘
```

#### 生成等待体验

**模态窗口（同步等待）**：
```
┌────────────────────────────────┐
│  正在生成 Workflow...          │
├────────────────────────────────┤
│  ✅ 分析操作意图               │
│  ⏳ 构建 MetaFlow (60%)        │
│  ⏸️  生成 Workflow              │
│                                │
│  [████████░░░░] 60%            │
│                                │
│  预计剩余时间：20 秒            │
└────────────────────────────────┘
```

**理由**：
- 30-60 秒可接受（用户理解 AI 需要时间）
- 明确进度和剩余时间（降低焦虑）
- 同步等待后立即可用（流畅体验）
