# AgentCrafter Demo 场景设计

**版本**: 1.0
**日期**: 2025-10-14
**目的**: 展示 Intent 复用和组合能力

---

## 🎯 设计原则

为了体现 **Intent 复用** 的价值，我们需要：

1. **部分重叠原则** - Workflow B 复用 Workflow A 的部分 Intent
2. **组合创新原则** - Workflow C = 复用的 Intent + 新的 Intent
3. **场景相关性** - 三个流程属于相同领域，用户能直观理解价值
4. **递进式复杂度** - A → B → C 逐步展示系统学习和复用能力

---

## 🎭 核心场景

**人物**: Alice，电商市场分析师
**业务目标**: 为自己的电商店铺做**选品决策** - 了解不同市场的热销商品和消费者偏好，找到高潜力商品

**业务价值**:
- 不是简单的"采集数据"，而是**发现商业机会**
- 从跨市场对比中找到"在美国热销但波兰市场未饱和"的选品机会
- 理解不同地区消费者的购买习惯差异（价格敏感度、品类偏好等）

**关键洞察**:
- **市场分析师的真实需求**: 不是"哪里买便宜"，而是"什么值得卖"
- **跨市场的复杂性**: 不同平台的热度指标不同（Amazon 的 Best Seller Rank vs Allegro 的销量/评论数）
- **可复用的商业智能**: 学会的"跨市场选品分析"能力，可以应用到任何品类、任何地区对比

## 📋 Part A: Workflow A - "采集商品基础信息"

### 场景描述

Alice 需要快速了解 Allegro 网站上咖啡产品的基本价格情况。

### 用户操作（录制过程）

1. 打开 Allegro.pl 网站
2. 在搜索框输入 "coffee"
3. 点击进入咖啡分类页面
4. 在商品列表页，滚动加载所有商品
5. 提取商品的**基础信息**：标题、销量统计、价格（仅列表页可见字段）
6. 导出数据到 CSV 文件

**自然语言描述**（用户输入）:
> "我需要从 Allegro 采集咖啡产品的价格信息，包括产品名称、价格和销量统计，保存到 CSV"

### 关键特征

- **操作简单**: 只在列表页操作，不需要点进详情页
- **数据浅层**: 只采集列表页直接可见的字段（2个字段）
- **学习内容**: 系统学会了网站导航、搜索、列表页提取、导出

### 演示重点

- 展示用户手动操作的录制过程
- 展示系统如何自动识别和命名 Intent
- 展示生成的 Workflow 结构
- 执行 Workflow，展示输出的 CSV 文件

## 📋 Part B: Workflow B - "采集 Amazon 商品数据"

### 场景描述

Alice 需要从 Amazon 采集同类咖啡产品数据，为后续的对比分析做准备。

### 用户需求（新网站采集）

**Alice 说**:
> "我还需要从 Amazon 采集咖啡产品数据，采集相同类型的信息：产品名称、价格、评分"

### 操作差异对比

| 维度         | Workflow A (Allegro) | Workflow B (Amazon) | 复用情况                   |
| ------------ | -------------------- | ------------------- | -------------------------- |
| 网站域名     | allegro.pl           | amazon.com          | **不同网站，不能复用**     |
| 打开网站     | ✓                    | ✓                   | **操作模式相同，可借鉴**   |
| 搜索商品     | ✓                    | ✓                   | **操作模式相同，可借鉴**   |
| 进入分类     | ✓                    | ✓                   | **页面结构不同，需新学习** |
| 提取商品列表 | ✓                    | ✓                   | **DOM结构不同，需新学习**  |
| 导出数据     | ✓                    | ✓                   | **完全复用（通用逻辑）**   |

### 关键差异

**跨网站的挑战**:
- **不同的 DOM 结构**: Amazon 的页面结构与 Allegro 完全不同
- **不同的 CSS 选择器**: 元素定位方式需要重新学习
- **不同的页面交互**: 某些操作的触发方式可能不同
- **相似的业务逻辑**: 但"搜索→分类→列表→提取→导出"的流程是相似的

**可复用的部分**:
- **高层操作模式**: "导航→搜索→提取→导出"的思路相同
- **数据导出逻辑**: CSV 导出是通用的，完全复用
- **操作经验**: 系统知道需要做什么，只是"怎么做"需要重新学习

### 系统展示的复用分析

```
🤖 Ami 正在分析您的需求...

✓ 检测到跨网站采集任务
✓ 虽然网站不同，但采集流程相似

可借鉴的操作模式:
  ≈ 网站导航模式（需调整 URL）
  ≈ 搜索操作模式（需定位新的搜索框）
  ≈ 数据提取模式（需识别新的 DOM 结构）

完全复用的部分:
  ✓ 数据导出逻辑（通用的 CSV 导出）

需要新学习的部分:
  + Amazon 页面元素定位
  + Amazon 分类页导航
  + Amazon 商品列表提取

🎯 复用价值:
  - 操作流程: 系统知道要"搜索→提取→导出"
  - 数据导出: 100% 复用
  - 页面操作: 需要重新学习（不同网站）

💡 系统理解了"电商数据采集"的通用模式，
   新网站只需要重新学习页面操作细节！
```

### 演示重点

- **对比展示**: 并排显示 Workflow A (Allegro) 和 Workflow B (Amazon)
- **结构相似性**: 展示两个 Workflow 的步骤顺序高度相似
  ```
  Workflow A: 导航→搜索→分类→提取→导出
  Workflow B: 导航→搜索→分类→提取→导出  (流程一致！)
  ```
- **颜色标注**:
  - 🟢 绿色: 完全复用的 Intent（数据导出）
  - 🟡 黄色: 可借鉴但需调整的 Intent（导航、搜索）
  - 🔵 蓝色: 新学习的 Intent（Amazon 特定操作）
- **生成速度**: 展示 Workflow B 生成速度（约 1-2 秒，因为有模式借鉴）
- **执行结果**: 展示 Amazon 商品数据的 CSV 文件

## 📋 Part C: Workflow C - "跨市场选品洞察分析"

### 场景描述

Alice 需要**做选品决策** - 通过对比波兰市场（Allegro）和美国市场（Amazon）的咖啡产品热度和消费者偏好，找到高潜力的选品机会。

**业务目标**:
- 识别"双市场验证"的安全选品（两边都热销）
- 发现"市场机会"（美国热销但波兰竞争少）
- 理解不同市场的消费者偏好差异（价格带、品类偏好）
- 生成可执行的选品建议和行动清单

**为什么这个场景有意义**:
- 不是简单的"价格对比"，而是**发现商业机会**的市场分析
- 体现了系统学习和复用**商业智能**（不只是数据采集）
- 展示"Amplifying Expertise" - Alice 的市场分析方法被系统学习，可以应用到任何品类

### 用户需求（跨域复用）

**Alice 说**:
> "我想对比波兰和美国市场的咖啡热销情况，分析消费者偏好差异，告诉我哪些产品值得进货，哪些是潜在机会"

### 任务分解

这个任务实际上包含三个部分：

1. **采集 Allegro 数据** - 可以直接复用 Workflow A ✅
2. **采集 Amazon 数据** - 可以直接复用 Workflow B ✅
3. **数据对比分析** - 需要新学习（新能力）❌

### 跨网站复用的特点

| 维度     | Allegro    | Amazon     | 复用情况                   |
| -------- | ---------- | ---------- | -------------------------- |
| 数据采集 | Workflow A | Workflow B | **100% 复用已有 Workflow** |
| 数据合并 | -          | -          | **新能力（首次实现）**     |
| 对比分析 | -          | -          | **新能力（未来可复用）**   |
| 报告生成 | -          | -          | **新能力（未来可复用）**   |

### 系统展示的组合策略

```
🤖 Ami 正在生成组合 Workflow...

复用已有 Workflow:
  ✓ Workflow A: 采集 Allegro 商品数据（完整复用）
  ✓ Workflow B: 采集 Amazon 商品数据（完整复用）

新增数据处理能力:
  + 合并数据集 - 标准化字段名（Allegro 和 Amazon 字段名不同）
  + 对比分析 - 计算价格差异、评分对比、识别最佳优惠
  + 生成报告 - 创建包含图表的 PDF 报告

🎯 复用价值:
  - 数据采集: 100% 复用 Workflow A + B
  - 数据处理: 100% 新学习（新能力，未来可复用）

💡 数据采集部分完全复用，只需要学习"如何对比分析"！
   这些分析能力未来可以用于任何跨网站对比场景。
```

### 分析层的业务价值

Workflow C 的核心价值不在于技术复杂度，而在于**业务洞察**：

- **输入**: 两个市场的商品数据（已通过 A、B 采集）
- **处理**: 跨市场对比分析（数据合并、消费者偏好分析、选品机会识别）
- **输出**: 可执行的选品决策报告
  - 哪些产品值得进货（"安全选品"）
  - 哪些是潜在机会（"美国热销但波兰竞争少"）
  - 不同市场的消费者偏好差异（价格带、品类偏好）

**为什么这个设计有意义**:
- 不只是"数据采集 + 数据处理"，而是**采集 + 商业智能**
- 体现"Amplifying Expertise" - Alice 的市场分析方法（如何识别选品机会、如何评估风险）被系统学习
- 这套分析逻辑可以复用到任何品类、任何市场对比场景

### 执行流程

1. **并行采集阶段**（同时进行）:
   - 左侧浏览器: 执行 Workflow A 采集 Allegro 数据
   - 右侧浏览器: 执行 Workflow B 采集 Amazon 数据

2. **数据处理阶段**（顺序进行）:
   - 合并两个数据集
   - 标准化字段名（Allegro 的字段可能是波兰语，Amazon 是英语）
   - 匹配同款产品（基于产品名称模糊匹配）
   - 计算对比指标（价格差异、评分对比）

3. **报告生成阶段**:
   - 创建对比表格
   - 生成价格对比柱状图
   - 生成价格-评分散点图
   - 标注最佳购买推荐
   - 导出 PDF 报告

### 演示重点

- **Workflow 复用展示**: 清晰标注 Workflow A 和 B 被完整复用
- **并行执行**: 两个浏览器同时采集数据，展示效率
- **数据合并**: 动画展示不同来源数据如何标准化和匹配
- **可视化报告**: 展示最终生成的 PDF 报告，包含：
  - 产品对比表格（产品名、Allegro价格、Amazon价格、价差、推荐）
  - 价格对比柱状图
  - 价格-评分散点图
  - 最佳购买推荐列表
- **未来价值**: 强调这些新能力可以复用到任何跨网站对比场景
  - 未来 Workflow D (eBay vs Allegro): 只需要学习 eBay 采集，对比分析直接复用
  - 未来 Workflow E (三站对比): 三个采集 Workflow 复用，对比逻辑扩展即可

## 💎 价值主张总结

### 1. 跨网站模式复用（A → B）

**场景**: 不同网站，相同采集需求

- **业务流程相似**: 导航→搜索→提取→导出（流程一致）
- **技术实现不同**: DOM 结构、页面元素定位需要重新学习
- **通用逻辑复用**: 数据导出完全复用

**价值**: 系统理解了"电商数据采集"的通用模式，新网站只需要学习页面操作细节

### 2. Workflow 级别复用（A + B → C）

**场景**: 组合已有能力，创造新价值

- **数据采集**: 100% 复用 Workflow A 和 B
- **数据处理**: 新增通用分析能力（未来可复用）

**价值**: Workflow 可以像乐高积木一样组合，快速构建复杂任务

### 3. 能力积累飞轮（A → B → C → 未来）

```
Workflow A: 学会 Allegro 电商采集
Workflow B: 学会 Amazon 电商采集 + 理解电商通用模式
Workflow C: 学会跨站对比分析

未来 Workflow D（eBay vs Allegro vs Amazon 三站对比）:
  ✓ eBay 采集: 需要新学习（约 5 秒）
  ✓ Allegro 采集: 复用 Workflow A
  ✓ Amazon 采集: 复用 Workflow B
  ✓ 三站对比: 扩展 Workflow C 的逻辑
  → 只需要学习 20%，80% 复用！

未来 Workflow E（同样的对比分析，应用到服装类目）:
  ✓ 采集逻辑: 复用 A、B 的模式（只需要调整类目）
  ✓ 对比分析: 100% 复用 Workflow C
  → 只需要学习 10%，90% 复用！
```

**价值**: 系统用得越多，学到的能力越多，新任务的学习成本越低

## 🎯 为什么这个设计能体现 Intent 复用价值？

### 1. Intent 的本质定义正确

- ❌ 错误: "搜索咖啡" vs "搜索茶叶" = 不同 Intent
- ✅ 正确: "列表页提取" vs "详情页提取" = 不同 Intent

**关键**: Intent 基于**操作模式**，而非**输入参数**

### 2. 复用的层次清晰

- **导航层**: 可复用（相同网站）
- **提取层**: 需新学习（不同深度/结构）
- **分析层**: 可复用（通用逻辑）

### 3. 价值递进明显

- **Workflow A**: 建立基础能力
- **Workflow B**: 在基础上深化（复用导航）
- **Workflow C**: 跨域扩展（复用处理逻辑）

### 4. 未来可扩展性强

学到的能力可以组合出无限可能：
- 新网站 × 已有能力 = 快速支持
- 新分析需求 × 已有数据处理 = 快速实现
