# AMI 产品路线图 - 对标 Eigent 完善计划

## 执行摘要

经过对 Eigent 代码库的深入分析和 AMI 现有实现的全面审查，本文档制定了一个系统性的产品完善计划。AMI 已经具备了多 Agent 协调、Memory Graph 学习系统等核心能力，但在用户体验、功能完整性和工具生态方面仍有提升空间。

---

## 一、现状对比总览

### 1.1 功能矩阵

| 功能模块 | Eigent | AMI 现状 | 差距等级 |
|---------|--------|---------|---------|
| **多 Agent 协调** | ✅ 8 种 Agent | ✅ 13 种 Agent | ✅ 超越 |
| **任务分解与调度** | ✅ Workforce | ✅ TaskOrchestrator | ✅ 相当 |
| **用户学习系统** | ❌ 无 | ✅ Memory Graph | ✅ 独特优势 |
| **工作流生成** | ✅ 基础 | ✅ Claude SDK | ✅ 相当 |
| **浏览器自动化** | ✅ 完整 | ✅ 完整 | ✅ 相当 |
| **实时任务可视化** | ✅ 优秀 | ⚠️ 基础 | 🔴 需加强 |
| **Agent 工作区展示** | ✅ 优秀 | ⚠️ 有框架 | 🔴 需加强 |
| **工具生态 (35+)** | ✅ 丰富 | ⚠️ 核心工具 | 🟡 需扩展 |
| **MCP 工具集成** | ✅ 动态加载 | ❌ 未实现 | 🔴 需实现 |
| **OAuth 集成管理** | ✅ 完整 UI | ⚠️ 后端有 | 🟡 需 UI |
| **历史回放** | ✅ 完整 | ⚠️ 有记录 | 🟡 需完善 |
| **终端输出展示** | ✅ xterm.js | ❌ 无 | 🔴 需实现 |

### 1.2 AMI 独特优势

1. **Memory Graph** - 用户操作学习和知识图谱，Eigent 完全没有
2. **录制回放系统** - 完整的用户操作录制和工作流生成
3. **本地优先架构** - 隐私保护，数据本地存储
4. **Claude Agent SDK** - 工作流生成使用最新 SDK
