## 4. Workflow 与执行

### 4.1 Workflow 列表

在 **Workflows** 区域，你可以看到所有已创建的 Workflow：

- 每条记录包含名称、状态和最近一次运行时间；
- 点击某条记录可以进入详情页面。

### 4.2 Workflow 详情

在 Workflow 详情页面，你通常可以：

- 查看 Workflow 的步骤和逻辑结构；
- 在支持的地方调整输入参数；
- 启动一次新的运行。

### 4.3 运行过程监控

启动运行后，可以打开 **实时执行 / 监控** 视图：

- 查看每一步的执行进度；
- 知道当前正在执行哪一步；
- 必要时停止或取消本次运行。

### 4.4 查看运行结果

运行结束后，你可以打开 **Execution Result（执行结果）**：

- 查看本次运行是否成功；
- 检查日志和输出数据；
- 根据结果返回 Workflow 做进一步调整。

### 4.5 AI 修改与脚本生成的等待时间

当你让 Ami 基于录制生成 Workflow，或者在已有 Workflow 上使用 AI 做修改时，应用通常需要一段时间来：

- 调用大模型，对 Workflow 结构进行重排或优化；
- 生成或更新辅助脚本，例如用于网页交互的 scraper agent。

在这段时间里，你可能会看到加载提示或「准备中」一类状态。实际耗时取决于：

- 录制或 Workflow 的长度和复杂度；
- 你连续发起了多少次「让 AI 再调整一下」的请求。

同样地，在启动 Workflow 执行时，Ami 也可能先花一点时间准备或更新脚本，然后才开始可见的步骤执行。这属于正常行为，你可以在监控或实时执行视图中观察整体进度。

### 4.6 使用 AI 对 Workflow 进行对话式修改

除了从录制或描述生成 Workflow 之外，你还可以通过 **对话** 的方式，请 AI 帮你查看和修改已有的 Workflow。

典型用法是：

1. **打开 Workflow 详情页面。**
2. 打开该 Workflow 的 AI 辅助编辑 / 对话面板（具体入口以实际界面为准）。
3. 在对话中做三件事：
   - 先让 AI 解释当前 Workflow 的逻辑，例如：「帮我解释一下这个 Workflow 每一步在做什么」。
   - 说明你期望的结果是什么，例如：「我希望除了价格以外，还把库存字段一起保存」。
   - 指出哪里执行错了，例如：「第二步在有些页面会报元素找不到的错误」。
4. 让系统给出修改方案。AI 可能会：
   - 修改一个或多个步骤（例如更新 scraper 步骤以抓取更多字段）；
   - 调整数据需求、条件判断或错误处理逻辑；
   - 建议你重新运行或对单一步骤进行测试。

你可以在这种对话式循环中迭代：

- 先运行或测试修改后的 Workflow；
- 如果仍然不符合预期，就把「实际结果」和「期望结果」告诉 AI；
- 视情况接受或继续调整 AI 提出的修改建议。
